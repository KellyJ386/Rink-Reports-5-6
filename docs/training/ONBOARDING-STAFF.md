# Staff Onboarding — Here's How to Do Your Job

*RinkReports · Version 5-6 · for staff*

This is the short version: just the things you'll actually do. Open RinkReports on your phone or tablet, sign in, and you'll see your menu (or the bottom tabs: **Home · Reports · Menu · Account**). You only see the modules your facility turned on and gave you access to. Tap a task below for the full how-to.

> **You only see your own facility's data — this is automatic.** Nothing to set up. If a menu item is missing, your facility either turned that module off or hasn't granted you access — ask your manager.

---

## Submit your reports

- [ ] **Daily Reports** — Tap **Daily Reports** → pick your **Work area** → pick the **Shift** → tick each task → optional **Note** → **Submit**. Each submit is its own record; you can't edit it after, so submit a fresh one if you need to correct something. *(See [Daily Reports §6 — Submit a daily report](./modules/daily-reports.md#submit-a-daily-report-staff).)*
- [ ] **Refrigeration** — Tap **Refrigeration** → set the **Reading time** → work down each section, filling in every reading → watch the **"Normal: …"** hints. If a value is **critically** out of range, fill in the **Corrective action** box → **Submit**. (The °F/°C toggle changes display only.) *(See [Refrigeration §6 — Log a refrigeration reading](./modules/refrigeration-logs.md#log-a-refrigeration-reading-staff).)*
- [ ] **Air Quality** — Tap **Air Quality** → choose the **Location** → enter each reading and watch the **range badge** (green = good). If it turns amber/red, read the banner, act, then write the required **Corrective action** note → **Submit**. *(See [Air Quality §6 — Record an air-quality reading](./modules/air-quality.md#record-an-air-quality-reading-staff).)*
- [ ] **Ice Operations** — Tap **Ice Operations** → pick the tab (**Ice Make / Edging / Blade Change / Circle Check**) → fill the fields → for a **Circle Check**, mark each item **Pass/Fail** and write a note on every **Fail** → **Submit**. *(See [Ice Operations §6](./modules/ice-operations.md#6-step-by-step-common-tasks).)*
- [ ] **Ice Depth** — Tap **Ice Depth** → tap a point on the rink diagram → type the depth (or use a Bluetooth caliper) → **Enter** moves to the next point → **Review & Submit** when done. *(See [Ice Depth §6 — Measure ice depth](./modules/ice-depth.md#measure-ice-depth-across-the-rink), and the [Bluetooth caliper steps](./modules/ice-depth.md#use-a-bluetooth-caliper).)*

## File an incident or accident

- [ ] **Incidents** — Tap **Incidents** → set **When did it happen?** → pick the **Facility space(s)** → write the **Description** → set **Severity**, toggle **Ambulance called** / **Follow-up required**, add **witnesses** if any → **Submit** and confirm. **You can edit your own report for 24 hours**, then it locks. No photos — describe it in words. *(See [Incident Reporting §6](./modules/incident-reporting.md#6-step-by-step-common-tasks).)*
- [ ] **Accidents** — File from the **Accidents** menu item the same way (reporter is filled in from your login). *(See the Incident chapter for the shared location list and review flow.)*

## Scheduling — self-service

Open **Scheduling** for your shifts and requests.

- [ ] **See your shifts** — The landing screen shows your **Next shift**, the week strip, and **Upcoming** shifts. *(See [Employee Scheduling §5 — Staff app](./modules/employee-scheduling.md#staff-app--reportsscheduling).)*
- [ ] **Set availability** — **Scheduling → Availability → New** → day, times, **Type** (Available / Preferred / Unavailable), optional area → **Add**. *(See [§6 — Submit availability](./modules/employee-scheduling.md#6-step-by-step-common-tasks).)*
- [ ] **Request time off** — **Scheduling → Time off → New request** → **Starts** / **Ends** / optional reason → **Submit**. Cancel your own while it's pending/approved.
- [ ] **Claim an open shift** — On the landing screen, find a shift under **Open · Pick up** → **Claim shift**. It may need manager approval. *(Online only.)*
- [ ] **Request a swap** — **Scheduling → Shift swaps → New swap request** → pick **your shift to give up**, optionally a coworker and their shift → **Send**. *(Online only.)*

## Working offline ("Save on this device")

RinkReports keeps working without a connection for **submitting reports**:

- When you're offline, the Submit button reads **"Save on this device"** (or "Save offline"). Tap it — your report is saved on your device and **submits automatically when you reconnect**. *(See any module's "Locking, saving & offline" section, e.g. [Daily Reports §8](./modules/daily-reports.md#8-locking-saving--offline).)*
- You'll see a pending count, and **Reports → Offline queue** lists anything still waiting to sync. You won't create duplicates.
- **What does NOT work offline:** claiming open shifts and shift swaps (those need a live connection). Submitting **availability** and **time-off** *does* queue offline. *(⚠ VERIFY — claim/swap are online-only; see [Employee Scheduling §8](./modules/employee-scheduling.md#what-queues-offline-vs-online-only).)*

## If something's stuck

- **A menu item is missing** → the module is off for your facility or you don't have access. Ask your manager.
- **"No permission" / "Account not set up"** → your login isn't fully set up or lacks the submit permission. Ask your manager.
- **You made a mistake after submitting** → you can't edit most reports (incidents give you 24 hours). Submit a corrected one, or ask a manager to add a follow-up note.

That's it — submit your reports, file incidents when needed, manage your own schedule. Each chapter linked above has a full walkthrough if you want more detail.
